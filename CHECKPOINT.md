# Checkpoint — перед следующим большим блоком

**Дата:** 15 марта 2026  
**Репозиторий:** https://github.com/rarawetara/MarTik

---

## Что сейчас стабильно работает

- **Phase 1:** Авторизация, профиль, ежедневные записи (`daily_entries`), главная (Home), страница «Сегодня» (Today), вода, задачи по дням (`daily_tasks`), навигация, Дневник, Настройки. Всё на русском.
- **Phase 2A:** Красота — продукты (`beauty_products`), карточки, фото в Storage (bucket `beauty-products`).
- **Phase 2B:** Рутины (`beauty_routines`, `beauty_routine_steps`), конструктор рутин, шаги с продуктами, мягкое удаление шагов.
- **Phase 2C:** Ежедневное выполнение рутин — `beauty_logs`, отметки на странице «Сегодня», сохранение по дням, история не перезаписывается.
- **Phase 2D:** Фото прогресса (`beauty_progress_photos`), загрузка лица/волос, галерея, Storage `beauty-progress`.
- **Phase 2E:** Метаданные фото прогресса — связь с рутиной, мягкие оценки (состояние кожи, волос, длина), редактирование.
- **Phase 2F:** Расписание рутин — `cadence_type`, `weekly_days`, `monthly_days`, `is_active`; на «Сегодня» показываются только due-рутины для выбранной даты.

Миграции: `001` … `008`. Логика `beauty_logs` и выполнения рутин не менялась в 2F — только фильтр по due.

---

## Правило перед следующим блоком

**Не трогать beauty-ядро без необходимости.**

- Не менять схему или логику: `beauty_logs`, `beauty_routine_steps`, ежедневное выполнение на «Сегодня».
- Новый функционал — новые таблицы/поля или отдельные модули, без переписывания существующей красоты.
