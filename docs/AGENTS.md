# Project cleanup rules

- Prefer small diffs.
- Do not change behavior unless explicitly requested.
- Do not rename public APIs without updating all usages.
- Prefer TypeScript-safe refactors.
- Avoid unnecessary abstraction.
- Remove dead code only when confidence is high.
- Keep UI text unchanged unless fixing obvious typos.
- Run lint after changes.
- Keep files modular, but do not split files just for the sake of splitting.
